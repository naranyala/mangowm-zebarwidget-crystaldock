# FLTK Panel: Bottom Dock with App Switcher

## Goal

Add a bottom dock to the FLTK panel that shows running applications and acts as a switcher (click to focus). Uses `wlr-foreign-toplevel-management` protocol to track windows.

## Context

- Current panel: `fltk-panel/panel.cpp` — regular xdg_toplevel window with launcher, workspaces, clock
- Protocol already available: `protocols/wlr-foreign-toplevel-management-unstable-v1-client.{h,c}`
- Production reference: `src/shells/fltk-panel/main.cpp` (raw C++, read-only taskbar)
- GTK reference: `src/gui/ocws-workspace-mgr.c` (full interactivity with `activate()` call)
- FLTK creates xdg_toplevel windows — not layer-shell, so this is an exploration prototype

### 2. `fltk-panel/toplevel-dock.cpp` — Implementation

Wayland protocol binding and window tracking:
- Bind `zwlr_foreign_toplevel_manager_v1` from registry
- Bind `wl_seat` for activation
- Manager listener: `.toplevel` → add window, `.finished` → cleanup
- Handle listener: `.title`, `.app_id`, `.state`, `.done`, `.closed`
- State decoding: parse `wl_array` for ACTIVATED/MINIMIZED/MAXIMIZED flags
- `activate()`: call `zwlr_foreign_toplevel_handle_v1_activate(handle, seat)`
- `close()`: call `zwlr_foreign_toplevel_handle_v1_close(handle)`

## Protocol Integration Details

### Binding from FLTK's Wayland display

FLTK's Wayland backend holds a `wl_display*`. We need to:
1. Get the display pointer (FLTK stores it internally)
2. Create a `wl_registry` and bind globals
3. Bind `zwlr_foreign_toplevel_manager_v1` (version 3)
4. Bind `wl_seat` (for activate calls)

### Window tracking flow

```
wl_registry_global("zwlr_foreign_toplevel_manager_v1")
  → bind manager
  → manager.toplevel → new handle → attach handle listener
    → handle.title("Firefox")     → store title
    → handle.app_id("firefox")    → store app_id
    → handle.state([ACTIVATED])   → set focused=true
    → handle.done                 → refresh dock
    → handle.closed               → remove from list
```

### Activation (focus switching)

```cpp
void ToplevelDock::activate(zwlr_foreign_toplevel_handle_v1 *handle) {
    if (handle && seat) {
        zwlr_foreign_toplevel_handle_v1_activate(handle, seat);
    }
}
```

This is the same call used in `ocws-workspace-mgr.c:368`.

## Dock UI Design

- **Position**: Bottom of FLTK window, 48px height
- **Layout**: Horizontal row of buttons, centered
- **Button style**: 48x48, rounded corners, icon+label
- **Visual states**:
  - Normal: Surface0 background, FG label
  - Focused: Accent background, bold label
  - Minimized: Muted background, dim label
- **Interactions**:
  - Left click: activate (focus) window
  - Right click: close window
  - Middle click: minimize window
